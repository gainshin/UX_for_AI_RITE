# UX for AI RITE

**Human-in-the-loop 以用戶為中心，打造負責任 AI 的最佳實踐**

本專案是「UX for AI」課程的綜合資源庫，旨在協助團隊系統化地將使用者體驗原則應用於人工智慧產品開發。我們強調價值導向的方法，在準確率與召回率之間取得平衡，並滿足使用者需求。

## 📚 專案概覽

**UX for AI RITE** 儲存庫是學習者與從業者的數位伴侶。它整合了方法論、案例研究、設計模式與工具，以支援「三線並行」的開發流程（UI/UX、AI 模型、數據/知識）。

### 主要功能

-   **案例庫 (`cases.html`)**：收錄 54+ 個實務方法，對應 AI 創新設計的五大階段（AI 意圖感知、AI 情境洞察、AI 概念探索、AI 方案驗證、AI 治理實現）。包含用於測試 AI 行為的「Spike」概念。
-   **設計模式牆 (`patterns.html`)**：可篩選的治理與部署策略資料庫。根據政策、風險控管與部署成熟度快速尋找合適的模式。
-   **工具庫 (`tools.html`)**：精選的研究、設計與治理工具列表，支援您的 AI 策略落地。
-   **學習者與管理者模式**：為學生 (`index.html`) 與講師 (`teacher.html`) 量身打造的專屬視圖。

## 🚀 核心概念

### RITE + Spike 流程
我們提倡 **RITE (Rapid Iterative Testing)** 與 **Spike** 概念，將 AI 模型視為需要持續檢驗的「病理切片」。
-   **Spike**：用於測試模型特定局部行為的小段程式碼。
-   **目標**：在完整 UI 實作前，驗證鏈式思考、護欄機制與安全策略。

### 三線並行開發 (Three-Track Parallel)
擺脫傳統線性瀑布流，我們提出同步迭代三大關鍵支柱：
1.  **台座 (UI/UX)**：展示模型能力的體驗舞台。
2.  **猴子 (AI Model)**：需要持續訓練與調校的核心智慧體。
3.  **莎士比亞 (Data)**：驅動模型學習的養分來源。

## 🛠️ 如何開始

在本地端執行此專案：

1.  **複製儲存庫**：
    ```bash
    git clone https://github.com/gainshin/UX_for_AI_RITE.git
    cd UX_for_AI_RITE
    ```

2.  **安裝依賴** (若適用，或直接啟動靜態伺服器)：
    ```bash
    npm install -g serve
    ```

3.  **啟動伺服器**：
    ```bash
    serve .
    ```

4.  打開瀏覽器並前往 `http://localhost:3000` (或 `serve` 指定的連接埠)。
